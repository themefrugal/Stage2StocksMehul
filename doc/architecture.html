🎯 What does this app do?
📊

Stage 2 Screener

Identifies stocks from the Nifty 750 universe that are breaking out of consolidation into an uptrend — the "Stage 2" phase described by Stan Weinstein. Scores each stock out of 7 based on price, volume, and moving average conditions.

🚀

Momentum Screener

Ranks stocks by risk-adjusted return (Sharpe ratio) across multiple timeframes — 3, 6, 9, and 12 months. Helps identify stocks with consistent, smooth upward momentum rather than volatile spikes.

💾

Smart Daily Cache

Results are stored once per trading day so the app is fast for repeated use. Data automatically refreshes after 7 PM IST each day, aligned with market close and NSE holidays.

📋 The 7-Point Stage 2 Scoring System
🟢 6–7 Strong Stage 2 🟡 4–5 Likely Stage 2 🟠 2–3 Early / Weak ⚪ 0–1 Not Stage 2
#ConditionWhat it checks
1Volume SurgeToday's volume is 2× or more of the recent average — institutional buying signal
2Higher HighToday's high exceeds the highest high of the past 50 days — price expanding upward
3Higher LowToday's low is above the lowest low of the past 50 days — floor rising
4MA50 RisingPrice is above 50-day average and that average itself is trending up
5MA200 RisingPrice is above 200-day average and that average is trending up — long-term health
6Above MA150Price above the 150-day average — medium-term trend confirmation
7MA StackMA50 > MA150 > MA200 — the "golden alignment" of moving averages
📡 Where does the data come from?
🌐

Yahoo Finance

Daily end-of-day (EOD) price and volume data for all stocks — free, no API key needed. Downloads up to 2 years of history at once.

📁

constituents.json

List of stock symbols for each Nifty index — Nifty 50, Next 50, Midcap 150, Smallcap 250, and Microcap 250. Updated manually as index rebalances occur.

📅

nse_holidays.json

NSE trading holiday calendar. Used to make sure the app always shows data for the last valid trading day, not a weekend or holiday.

📂 Data Flow by Screener

Stage 2 Screener

Universe: Full Nifty 750 (hardcoded)
History: 2 years of OHLCV
Cache: File-based JSON, keyed by trading date
Filter timing: Filtering applied in-memory after load (instant)
Re-fetch: Once per trading day; fallback to older cache on failure

Momentum Screener

Universe: User-selected (8 options)
History: 18 months of OHLCV
Cache: In-memory only, 1-hour TTL (no disk persistence)
Filter timing: All filters applied to cached DataFrame in real time
Re-fetch: On button click after cache expires

🧩 Function Reference
load_nse_holidays()
util
Loads nse_holidays.json and returns a set of date strings (YYYY-MM-DD). Handles both "tradingDate" and "tradingDate " (trailing space) key variants.
get_last_valid_trading_date(start, holidays)
util
Walks backward from start (max 10 days) skipping weekends and holidays. Returns the nearest valid NSE trading date as a string.
load_json_cache(key) / save_json_cache(df, key)
cache
Read/write daily_cache/{key}.json as a pandas DataFrame using orient="records". Returns None on any read failure.
_rsi_wilder(series, period=14)
indicator
Wilder's RSI using EWM with alpha=1/period. Returns a pd.Series of RSI values. Used only in Stage 2 scoring.
score_stage2(df)
scoring
Takes an OHLCV DataFrame (min 250 rows). Computes MA50/150/200, RSI, volume ratio. Returns a dict with Score (0–7), Stage label, Close, Vol_Ratio, RSI, MA values, Illiquid flag, and Avg_Vol — or None if data insufficient.
_count_circuits(df)
momentum
Counts days where pct_change ≈ ±5%, ±10%, or ±20% within ±0.1% tolerance. Returns an integer circuit count.
_calculate_sharpe(df, period_days)
momentum
Annualized Sharpe = Annualized RoC ÷ Annualized Std Dev over the last period_days trading days. Returns None if insufficient data or zero std dev.
_calculate_positive_days_pct(df, months)
momentum
Calculates % of days where Close > prev Close over approx months × 21 trading days. Returns a float or None.
score_momentum(df)
momentum
Calls all momentum sub-functions and returns a dict with: Close, 52w_High, DMA100, DMA200, Vol_Median, 1Y_Change, Pct_From_52W_High, Circuit_Count, Sharpe_3M/6M/9M/1Y, Volatility, Pos_Days_3M/6M/12M.
_get_universe_symbols(universe, constituents)
data
Maps composite universe names (e.g. "Nifty LargeMidcap 250") to their constituent index lists, then flattens and deduplicates symbols.
fetch_full_universe(rsi_filter, for_momentum)
data
Bulk downloads all 750 tickers via yf.download(threads=True). Iterates per-ticker to extract OHLCV sub-frames, dispatches to score_stage2 or score_momentum. Handles both single and multi-ticker MultiIndex column layouts.
fetch_momentum_universe(universe)
data
Fetches only the symbols for a user-selected universe. Includes robust multi vs. single ticker column flattening and skips tickers not present in the returned MultiIndex.
resolve_screener_data(rsi_filter, for_momentum, universe)
orchestrator
Main cache resolution pipeline. Applies 7 PM IST cutoff logic → resolves trading date → checks disk cache → fetches if miss → saves → falls back to older cache. Returns (df, date_str, is_cached).
get_momentum_full_universe_data()
cache
Module-level in-memory cache (_momentum_full_universe_cache dict). Fetches full-universe momentum data once per hour. Called by momentum_screener_ui() on every Run click.
_calculate_avg_sharpe(row, method)
momentum
Row-wise helper for sorting. Reads Sharpe_3M/6M/9M/1Y fields from a DataFrame row and returns the appropriate single or averaged value based on the sort_method string.
stage2_screener_ui() / momentum_screener_ui()
ui
Full Streamlit UI functions. Each renders a 1:4 sidebar/main column layout. Use Streamlit session_state keys prefixed stage2_ / mom_ to avoid cross-tab key collisions.
main()
entrypoint
Creates st.tabs() and calls stage2_screener_ui() / momentum_screener_ui(). Called at module level; Streamlit re-runs this on every interaction.
🔑 Key Implementation Notes
MultiIndex Column Handling When yfinance returns data for multiple tickers, columns are a MultiIndex: (ticker, OHLCV-field). For a single ticker, columns are still MultiIndex but structured as (ticker, field). The code handles both cases differently per-function — fetch_full_universe uses c[0] extraction, while fetch_momentum_universe uses c[1] for single-ticker case and direct raw[t] for multi-ticker.
Session State Keys All Streamlit widget keys are namespaced: stage2_* for Stage 2 tab, mom_* for Momentum tab. The Stage 2 run trigger uses st.session_state["stage2_run_triggered"] to persist across reruns; Momentum is stateless (only runs on button click each time).
⚠️ Known Quirks • HISTORY_PERIOD_MOMENTUM is assigned twice in fetch_full_universe() (line 291 is a duplicate — safe but redundant).
• Stage 2 cache applies RSI filter before saving — re-running with RSI toggle off on a cached day will still show pre-filtered data. Filter is re-applied in UI for display but the cache itself may be pre-filtered.
• Momentum universe filter in momentum_screener_ui() uses isin([selected_universe]) which matches against the Index column (individual index names, not composite names like "Nifty LargeMidcap 250") — composite universes work because the data is fetched from expanded symbol lists upstream.
